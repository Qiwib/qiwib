Installation

QiwiB has been written and tested in Linux operating systems with Fedora 13 and 14, Ubuntu 10.04, PCLinuxOS 2010 and OpenSuse 11.4 installed. Also, QiwiB works for MacOSX, however it has not been thoroughly tested yet.
QiwiB has been tested for GNU Octave versions 3.2.3 to 3.4.1 (old versions) and 3.4.2 (current stable). Usually the big Linux distributions provide binary packages of octave and octave-forge. If there are no packages available or different versions of octave than provided are needed one can follow the instructions below. MacOS users might want to download the octave packages from MacPorts (recommended), Fink repositories or Octave Forge directly.

Instructions for installing octave binary packages

All big Linux distributions include octave as a binary package and can easily be installed from your distribution's package manager. The following packages are needed (names might vary depending on your distribution):
  • octave
  • octave-headers
  • octave-odepkg
  • octave-general
MacOS Users can download a copy from Octave Forge or use the Fink or MacPorts (recommended) repositories. The installation in MacOS may produce errors that are unrelated to QiwiB and result due to minor differences between the GNU tools in MacOS and current Linux distributions. Usually an installation of Octave via Fink or MacPorts seems to resolve those issues automatically. In order to install the odepkg and general packages please follow the instructions below in "Compile additional Octave packages".

Instructions for compiling octave from source

  • Download Octave from here
  • Install according to the instructions in the download file
Octave will remind you of missing dependencies, i.e. additional libraries or source code.
Good introductions to octave programming are given on the GNU Octave webpage.

Compile additional Octave packages

  • Download additional packages odepkg and general from here
  • Start octave from command line and install packages with 'pkg install [packagename]'
OR (for Octave >= 3.4.0 with internet connection)
  • Start octave from command line and install packages with 'pkg install -forge [packagename]', where [packagename] is odepkg or general.
    This will automatically download the packages and install them (if download fails please do the manual method above).
Octave will remind you of missing dependencies, i.e. additional libraries or source code. Furthermore, other packages might be needed to be installed as additional dependencies. The installer will print a message if this is the case. Install these packages first before you continue to install odepkg and general.

QiwiB

Once Octave is installed and working properly start the setup process for qiwib by executing the Linux command line command 'sh [path to setup.sh]/setup.sh'. This starts the setup script that can be found in the bin/ directory of the QiwiB installation. Follow the instructions. It will ask you for the location of the octave binary as it can be possible to have multiple instances of octave installed. After that you can test if the installation has been successful by starting qiwib from the bin/ directory on the Linux command line.
Alternatively the setup script can append the following two lines to ~/.bashrc to set PATH to all QiwiB scripts (The QiwiB scripts from the bin/ directory can be executed from any folder without specifying the full path):

echo PATH=$setup_path:"\$PATH"
echo "export PATH"

This can also be done manually after the installation.

Additional tools

Furthermore, QiwiB's movie feature needs the following programs to be installed (you can check from the command line with which convert and which mencoder if those programs are properly already installed):
  • ImageMagick (can be downloaded from here or installed from a package in your favourite Linux distribution)
  • mencoder (can be downloaded from here or installed from a package in your favourite Linux distribution). Also, all necessary codecs to play the movie files should be included in the mplayer package.
MacOS users can download these programs from here and here as well or alternatively from the Fink or MacPorts repositories.
